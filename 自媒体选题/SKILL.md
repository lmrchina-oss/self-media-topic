# 自媒体选题

## 任务目标
- 本 Skill 用于：追踪指定领域和平台的热点话题，多维度评估选题潜力，提供针对性的标题优化建议
- 能力包含：热点搜索、选题评分、平台适配、标题优化、执行建议
- 触发条件：用户需要选择内容方向、分析热点趋势、优化平台标题或制定内容策略

## 操作步骤

### 1. 解析用户输入
从用户请求中提取以下信息：
- **领域关键词**：目标领域（如 AI工具、美妆、健身等）
- **目标平台**：分析哪些平台（小红书、B站、公众号）
- **时间范围**：热点时间范围（今天、近一周、近一个月、近三个月）
- **选题数量**：推荐多少个选题（默认5个）

### 2. 搜索热点数据
使用内置搜索功能获取相关热点：
- 构建搜索查询："{关键词} {时间范围} 热点 新闻 趋势 {平台}"
- 分析搜索结果，提取潜在选题
- 记录每个选题的热度、讨论度、时间信息

### 3. 多维度评分
基于 [references/scoring_criteria.md](references/scoring_criteria.md) 对每个选题进行评分：

**6个维度（总分100分）：**
1. **热度趋势**（25分）：热榜表现、搜索趋势、讨论热度
2. **时间匹配度**（20分）：时间敏感性、时效性价值
3. **平台适配度**（20分）：根据目标平台的用户画像和内容偏好评估
4. **内容稀缺性**（15分）：竞争度分析、差异化空间
5. **可执行性**（10分）：制作难度、资源需求
6. **商业价值**（10分）：变现潜力、长期价值

**评分要点：**
- 真实优先：基于实际数据评分，不夸大
- 平台差异化：不同平台的评分标准有所侧重
- 实用性至上：确保推荐的是用户能实际执行的选题
- 避免同质化：优先推荐有差异化空间的选题

### 4. 筛选和排序
- 按总分从高到低排序
- 选出Top {选题数量} 个选题
- 确保每个选题在各个维度都有明确得分理由

### 5. 优化平台标题
基于 [references/platform_guidelines.md](references/platform_guidelines.md) 为每个选题优化标题：

**小红书标题公式：** 疑问式 + 利益点 + emoji + 关键词
- 字数：20-30字
- 特征：实用性强、情感共鸣、信息密度高

**B站标题公式：** 悬念/反差 + 数据/数字 + 垂直领域 + 情绪词
- 字数：25-40字
- 特征：内容深度、创意独特、互动性强

**公众号标题公式：** 数字 + 痛点 + 解决方案 + 紧迫感
- 字数：20-35字
- 特征：观点犀利、数据支撑、情绪共鸣

为每个平台提供3个标题建议，并说明优化理由。

### 6. 给出执行建议
为每个选题提供具体可执行的创作建议：
- **发布时间**：建议的时间段和原因
- **内容形式**：图文/视频/长文等
- **关键元素**：必须包含的关键元素
- **注意事项**：制作过程中需要注意的点

### 7. 格式化输出
严格按照 [references/output_template.md](references/output_template.md) 的格式输出结果。

## 使用示例

**用户请求：**
"帮我分析AI工具领域的小红书和公众号热点，近一周时间，推荐5个选题"

**执行流程：**
1. 搜索："AI工具 近一周 热点 新闻 趋势 小红书 公众号"
2. 识别潜在选题：如"AI写作工具对比"、"AI设计工具测评"等
3. 基于6维度评分，筛选Top 5
4. 为小红书和公众号各优化3个标题
5. 给出每个选题的执行建议
6. 按标准格式输出结果

## 注意事项
- 仅在需要时详细读取参考文档，保持上下文简洁
- 评分时要基于真实数据，避免主观臆断
- 标题优化要符合各平台特征，避免模板化
- 执行建议要具体可操作，避免空泛建议
- 充分利用智能体的语言理解和生成能力，专注于策略和创意
